MCPVault Documentation

Welcome to the MCPVault documentation. Here you'll find everything you need to secure your Model Context Protocol infrastructure.

Installation

MCPVault can be installed via npm, Docker, or from source. Choose the method that best fits your deployment environment.

NPM Installation

# Install globally
npm install -g @mcpvault/cli

# Or add to your project
npm install @mcpvault/sdk

Docker Installation

# Pull the latest image
docker pull mcpvault/server:latest

# Run with default configuration
docker run -p 8080:8080 mcpvault/server:latest

Quick Start

Get MCPVault up and running in minutes with our quick start guide.

  1. Initialize MCPVault in your project directory:
mcpvault init
  1. Configure your compliance requirements:
mcpvault config set compliance.frameworks soc2,hipaa
  1. Start the MCPVault proxy:
mcpvault start

Configuration

MCPVault uses a YAML configuration file for all settings. Here's a complete example:

# mcpvault.yaml
version: 1.0
server:
  host: 0.0.0.0
  port: 8080
  tls:
    enabled: true
    cert: /path/to/cert.pem
    key: /path/to/key.pem

auth:
  providers:
    - type: oauth2
      issuer: https://auth.example.com
    - type: api_key
      header: X-API-Key

compliance:
  frameworks:
    - soc2
    - hipaa
    - iso27001
  
monitoring:
  enabled: true
  metrics_port: 9090
  logs:
    level: info
    format: json

Authentication

MCPVault supports multiple authentication methods to secure your MCP servers.

API Key Authentication

curl -H "X-API-Key: your-api-key" \
     https://api.mcpvault.com/v1/servers

OAuth 2.0

curl -H "Authorization: Bearer your-jwt-token" \
     https://api.mcpvault.com/v1/servers

Server Management

Manage your MCP servers through the MCPVault API.

List Servers

GET /v1/servers

Response:
{
  "servers": [
    {
      "id": "srv_123",
      "name": "production-mcp",
      "endpoint": "mcp://prod.example.com:8443",
      "status": "healthy",
      "compliance": ["soc2", "hipaa"]
    }
  ]
}

Register New Server

POST /v1/servers
Content-Type: application/json

{
  "name": "staging-mcp",
  "endpoint": "mcp://staging.example.com:8443",
  "compliance_requirements": ["soc2"]
}

Monitoring

Monitor all MCP server interactions in real-time.

Metrics Endpoint

Prometheus-compatible metrics are exposed at /metrics:

# HELP mcpvault_requests_total Total number of requests
# TYPE mcpvault_requests_total counter
mcpvault_requests_total{server="prod-mcp",status="success"} 12453

# HELP mcpvault_request_duration_seconds Request duration
# TYPE mcpvault_request_duration_seconds histogram
mcpvault_request_duration_seconds_bucket{le="0.1"} 8953

Compliance Setup

Configure MCPVault for your compliance requirements.

SOC2 Configuration

mcpvault compliance enable soc2
mcpvault compliance configure soc2 \
  --audit-retention 7y \
  --encryption aes-256-gcm \
  --access-reviews quarterly

HIPAA Configuration

mcpvault compliance enable hipaa
mcpvault compliance configure hipaa \
  --phi-encryption required \
  --access-logs detailed \
  --breach-notification enabled

Best Practices

Follow these best practices for optimal security and performance.

Security Best Practices

  • Always use TLS for all connections
  • Rotate API keys regularly (recommended: every 90 days)
  • Enable audit logging for all production environments
  • Use separate credentials for each environment
  • Implement rate limiting to prevent abuse

Performance Optimization

  • Use connection pooling for database connections
  • Enable caching for frequently accessed data
  • Monitor resource usage and scale accordingly
  • Use regional deployments for global applications
